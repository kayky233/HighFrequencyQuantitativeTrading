# HFQT 用户手册

版本：当前开发版  
适用范围：本地运行的 HFQT 实时量化看板、CLI、模拟盘与本地模型分析链路

## 1. 产品简介

HFQT 是一个本地运行的事件驱动量化交易系统，目标是把以下链路串起来：

```text
新闻/舆情/行情抓取
-> 特征提取
-> 模型分析
-> 复核
-> 风控
-> 模拟执行或券商执行
-> 持仓/盈亏/日志审计
```

当前版本支持：

- 本地网页看板
- `local_paper` 本地模拟盘
- `futu_sim` 适配骨架
- `uSmart Open API` 适配骨架
- 本地或远程 OpenAI-compatible 模型
- 多 Agent 配置：
  - 抓取 Agent
  - 分析 Agent
  - 复核 Agent
  - 风控 Agent

## 2. 当前运行模式

系统目前有三类数据/执行来源：

- 情报抓取：真实公网抓取
  - `Google News RSS`
  - `Yahoo Finance News`
  - `xreach`
  - 指定 X 账号监控
  - `Whale Alert` 链上大额异动
  - `BTC ETF flow` 结构化数据
  - `Fed / CPI / 非农` 宏观事件流
  - 可选 `Alpha Vantage`
- 行情/K 线：真实公网数据
  - `Yahoo chart`
  - `yfinance`
- 交易执行：
  - 默认是 `local_paper` 本地模拟盘
  - 后续可切 `futu_sim`
  - 再后续可切 `uSmart`

这意味着：

- 新闻和行情很多是真实抓取
- 当前默认成交和盈亏仍是模拟
- 如果你把分析模式切到 `llm`，分析、复核、风控都可以交给本地模型

## 3. 安装与启动

### 3.1 安装依赖

在项目根目录执行：

```powershell
pip install -e .
```

如果你不想用 editable mode，也可以手动安装最小依赖：

```powershell
pip install fastapi uvicorn pydantic httpx
```

### 3.2 启动 Web 服务

推荐：

```powershell
$env:PYTHONPATH="src"
python -m hfqt.app serve-api --host 127.0.0.1 --port 8011
```

然后打开：

```text
http://127.0.0.1:8011/
```

### 3.3 一键脚本

常用脚本在 [scripts](e:\HighFrequencyQuantitativeTrading\scripts)：

- [ignite_local.ps1](e:\HighFrequencyQuantitativeTrading\scripts\ignite_local.ps1)：本地模拟盘点火
- [ignite_analysis.ps1](e:\HighFrequencyQuantitativeTrading\scripts\ignite_analysis.ps1)：分析优先模式
- [ignite_public_no_key.ps1](e:\HighFrequencyQuantitativeTrading\scripts\ignite_public_no_key.ps1)：免 Key 公网数据点火
- [ignite_futu.ps1](e:\HighFrequencyQuantitativeTrading\scripts\ignite_futu.ps1)：Futu 模拟盘点火
- [demo_trading_today.ps1](e:\HighFrequencyQuantitativeTrading\scripts\demo_trading_today.ps1)：演示当日交易

## 4. 配置方式

### 4.1 推荐方式

系统推荐使用本地 `.env` 文件配置。

配置模板见：

- [.env.example](e:\HighFrequencyQuantitativeTrading\.env.example)
- [.env.usmart.example](e:\HighFrequencyQuantitativeTrading\.env.usmart.example)

系统启动时会优先读取：

- `HFQT_ENV_FILE` 指定的文件
- 如果没指定，默认读取项目根目录下的 `.env`

### 4.2 配置中心

网页左侧提供了“系统配置中心”：

- 默认折叠
- 按分组展示参数
- 支持搜索
- 支持生成 `.env` 草稿
- 支持复制草稿

注意：

- 这里编辑的是浏览器本地草稿
- 不会直接修改服务器上的 `.env`
- 敏感值不会回显旧值

使用方式：

1. 打开“系统配置中心”
2. 搜索或展开相应分组
3. 修改参数
4. 点击“生成 .env 草稿”
5. 将草稿复制到你的本地 `.env`
6. 重启服务

## 5. 关键配置项说明

### 5.1 基础运行

- `HFQT_ENV_FILE`
  - 指定环境变量文件路径
  - 默认 `.env`
- `HFQT_API_HOST`
  - Web 服务监听地址
- `HFQT_API_PORT`
  - Web 服务端口
- `HFQT_DATABASE_PATH`
  - SQLite 数据库路径

### 5.2 交易与风控

- `HFQT_DEFAULT_BROKER`
  - 默认执行通道
  - 可选：`local_paper`、`futu_sim`、`usmart`
- `HFQT_ALLOWED_SYMBOLS`
  - 白名单标的池
- `HFQT_AUTO_TRADE_ENABLED`
  - 是否默认自动执行
- `HFQT_AUTO_TRADE_COOLDOWN_MINUTES`
  - 同一标的自动交易冷却时间
- `HFQT_MAX_NOTIONAL_PER_ORDER`
  - 单笔最大名义金额
- `HFQT_MAX_ORDERS_PER_DAY`
  - 单日最大订单数
- `HFQT_MIN_CONFIDENCE`
  - 最低置信度

### 5.3 模型与 Agent

主分析模型：

- `HFQT_LLM_BASE_URL`
- `HFQT_LLM_API_KEY`
- `HFQT_LLM_PROVIDER`
- `HFQT_LLM_MODEL_PRIMARY`

### 5.4 新闻与数据时效

- `HFQT_NETWORK_INTEL_LIMIT`
  - 每次抓取的情报数量上限
- `HFQT_NETWORK_INTEL_PRIMARY_AGE_HOURS`
  - 优先使用的最新消息窗口，默认建议 `2h`
- `HFQT_NETWORK_INTEL_MAX_NEWS_AGE_HOURS`
  - 普通新闻硬上限，默认建议 `4h`
- `HFQT_NETWORK_INTEL_MAX_SOCIAL_AGE_HOURS`
  - 社媒硬上限，默认建议 `2h`
- `HFQT_NETWORK_INTEL_IMPORTANT_EVENT_AGE_HOURS`
  - 重大事件参考窗口，仅战争、制裁、破产等重大消息可超出常规窗口保留为参考
- `HFQT_NETWORK_INTEL_IGNORE_QUERY`
  - 忽略抓取关键词
- `HFQT_SCAN_CACHE_TTL_SECONDS`
  - 扫描缓存秒数
- `HFQT_SCAN_INCREMENTAL_TTL_SECONDS`
  - 增量刷新秒数
- `HFQT_BTC_PROXY_SYMBOLS`
  - 定义哪些标的会额外接入 BTC 专属情报层
- `HFQT_X_MONITOR_ENABLED`
  - 是否开启指定 X 账号监控
- `HFQT_X_MONITOR_ACCOUNTS`
  - 需要重点监控的 X 账号名单，写 handle，不带 `@`
- `HFQT_X_MONITOR_POSTS_LIMIT`
  - 每轮最多抓多少条指定账号帖子
- `HFQT_XREACH_ENABLED`
  - 是否开启全网关键词搜索
- `HFQT_WHALE_ALERT_ENABLED`
  - 是否开启 Whale Alert / 链上大额转账监控
- `HFQT_WHALE_ALERT_HANDLE`
  - 默认抓取的 Whale Alert 账号
- `HFQT_WHALE_ALERT_MIN_BTC`
  - 只保留多少 BTC 以上的大额转账
- `HFQT_WHALE_ALERT_MIN_USD`
  - 只保留多少 USD 以上的大额转账
- `HFQT_BTC_ETF_FLOW_ENABLED`
  - 是否开启 11 只 BTC ETF 净流入/流出抓取
- `HFQT_BTC_ETF_FLOW_JINA_URL`
  - BTC ETF flow 当前使用的数据入口
- `HFQT_BTC_ETF_FLOW_LOOKBACK_DAYS`
  - ETF flow 趋势回看窗口
- `HFQT_MACRO_EVENT_ENABLED`
  - 是否开启 Fed / CPI / 非农事件流
- `HFQT_MACRO_FED_RSS_URL`
  - Fed 官员讲话 RSS
- `HFQT_MACRO_CPI_RELEASE_URL`
  - CPI 发布摘要入口
- `HFQT_MACRO_NFP_RELEASE_URL`
  - 非农发布摘要入口
- `HFQT_MACRO_EVENT_LIMIT`
  - 每轮最多带入多少条宏观事件
- `HFQT_SETTLEMENT_WINDOWS`
  - 结算窗口日历配置（JSON 列表）
- `HFQT_DECISION_LOG_FILENAME`
  - 决策链日志文件名

## 6. 本地模型配置示例

如果你已经有本地 OpenAI-compatible 模型服务，可以在 `.env` 中这样写：

```env
HFQT_LLM_BASE_URL=http://127.0.0.1:8001/v1
HFQT_LLM_API_KEY=dummy
HFQT_LLM_PROVIDER=deepseek
HFQT_LLM_MODEL_PRIMARY=deepseek-chat
HFQT_LLM_MODEL_FALLBACK=deepseek-chat
HFQT_LLM_MODE=primary

HFQT_INTEL_AGENT_ENABLED=true
HFQT_INTEL_AGENT_MODEL=deepseek-chat

HFQT_REVIEW_AGENT_ENABLED=true
HFQT_REVIEW_AGENT_MODEL=deepseek-chat

HFQT_RISK_AGENT_ENABLED=true
HFQT_RISK_AGENT_MODEL=deepseek-chat
```

如果四个 Agent 共用同一个本地端点：

- `BASE_URL` 可以只填主模型的
- 其他 agent 的 `BASE_URL / API_KEY` 可留空
- 它们会自动继承主模型配置

### 6.1 客户机 Ollama / Qwen 示例

如果客户机本地跑的是 `Ollama`，并且测试脚本已经像下面这样成功：

- `base_url=http://localhost:11434/v1`
- `api_key=ollama`
- `model=qwen2.5:14b`

那么 HFQT 里可以直接按下面配置：

```env
HFQT_LLM_BASE_URL=http://127.0.0.1:11434/v1
HFQT_LLM_API_KEY=ollama
HFQT_LLM_PROVIDER=ollama
HFQT_LLM_MODEL_PRIMARY=qwen2.5:14b
HFQT_LLM_MODE=primary

HFQT_INTEL_AGENT_ENABLED=true
HFQT_INTEL_AGENT_PROVIDER=ollama
HFQT_INTEL_AGENT_BASE_URL=http://127.0.0.1:11434/v1
HFQT_INTEL_AGENT_API_KEY=ollama
HFQT_INTEL_AGENT_MODEL=qwen2.5:14b

HFQT_REVIEW_AGENT_ENABLED=true
HFQT_REVIEW_AGENT_PROVIDER=ollama
HFQT_REVIEW_AGENT_BASE_URL=http://127.0.0.1:11434/v1
HFQT_REVIEW_AGENT_API_KEY=ollama
HFQT_REVIEW_AGENT_MODEL=qwen2.5:14b

HFQT_RISK_AGENT_ENABLED=true
HFQT_RISK_AGENT_PROVIDER=ollama
HFQT_RISK_AGENT_BASE_URL=http://127.0.0.1:11434/v1
HFQT_RISK_AGENT_API_KEY=ollama
HFQT_RISK_AGENT_MODEL=qwen2.5:14b
```

说明：

- `HFQT_LLM_PROVIDER=ollama` 主要用于界面显示和配置对齐。
- `HFQT_LLM_API_KEY=ollama` 只是兼容 OpenAI SDK 的占位值。
- 如果四个 Agent 共用同一个本地端点，就把四组 `BASE_URL / API_KEY / MODEL` 全部对齐即可。
- 如果你希望模型直接决定数量和价格，提示词里应始终返回：
  - `action`
  - `confidence`
  - `quantity`
  - `limit_price`
- 建议规则：
  - `BUY/SELL` 时 `quantity > 0`
  - `HOLD` 时 `quantity = 0`
  - `limit_price` 无法确定时可返回 `null`

如果模型只返回了 `action + confidence`，系统也能运行：

- 数量会回退到仓位引擎
- 价格会回退到事件价格或行情快照价格

## 7. 界面使用说明

### 7.1 左侧区域

左侧主要用于配置和触发：

- Owner Token
- 系统配置中心
- 观察列表
- 手工标的 / 关键词
- 数量 / 情绪 / 价格
- 标题 / 正文
- 分析模式
- 执行通道
- 模型档位
- 自动刷新
- 自动交易

### 7.2 分析模式

- `MockReasoner`
  - 不走模型
  - 用固定因子逻辑生成意图
- `DeepSeek / LLM`
  - 抓取、分析、复核、风控都可走模型

如果本地模型已可用，建议默认使用 `llm`。

### 7.3 模型档位

- `primary`
  - 主力模型
- `fallback`
  - 保底模型
- `auto`
  - 自动根据输入长度或失败重试切到 fallback

### 7.4 右侧看板

右侧主要显示：

- 当前交易目标
- 信号与风控
- 最新订单
- 模型与刷新
- 组合盈亏
- 收益拆解
- 交易曲线与买卖点
- 观察列表候选
- 最近分析与订单
- 原始结果 JSON

默认折叠的区域可以按需展开。

## 8. 推荐操作流程

### 8.1 日常使用

1. 配好本地 `.env`
2. 启动 Web 服务
3. 打开看板
4. 确认：
   - `analysis mode = llm`
   - `provider / agents` 显示正确模型
   - `default broker` 正确
5. 观察候选标的
6. 检查：
   - 新闻是否够新
   - 曲线是否确认
   - 风控状态是否允许
7. 再决定是否开启自动交易

### 8.2 手工验证

如果你想先验证模型是否会给出买卖判断：

1. 在左侧填入：
   - 标的
   - 标题
   - 正文
   - 情绪分数
   - 参考价格
   - 数量
2. 选择 `DeepSeek / LLM`
3. 点击“提交手工事件”
4. 在右侧看：
   - action
   - quantity
   - confidence
   - review
   - risk

### 8.3 公网真实抓取验证

1. 填写标的和抓取关键词
2. 点击“抓取并分析最新情报”
3. 检查候选卡片中的：
   - 新闻标题
   - 新闻链接
   - 发布时间
   - 曲线确认
   - 触发摘要

## 9. 自动交易说明

自动交易只在满足以下条件时才会触发：

- `auto trade` 已开启
- 候选进入 `execute` 桶
- 执行风控允许
- 未命中同标的冷却时间
- 未超过单轮最大自动下单数

当前建议：

- 先关闭自动交易做观察
- 确认本地模型判断稳定后再逐步打开

## 11. Owner Control

如果你希望系统写操作和交易权限只由你控制，可以配置：

```env
HFQT_OWNER_CONTROL_ENABLED=true
HFQT_OWNER_TOKEN=your-private-token
HFQT_OWNER_SESSION_TOKEN=your-private-token
```

启用后：

- 敏感写操作需要 `X-HFQT-Owner-Token`
- 你可以随时锁交易
- 交易锁状态会持久化保存

## 13. 常见问题

### 13.1 页面刷新后内容变空

先用 `Ctrl+F5` 强刷。  
系统会保存大部分页面状态，但浏览器缓存旧脚本时可能会看到旧页面。

### 13.2 20 秒到了还是没刷新

不一定是坏了。首轮全观察池扫描可能较慢，尤其在：

- 观察列表很大
- 抓取源很多
- LLM 开启
- 网络较慢

可通过以下参数调优：

- `HFQT_SCAN_CACHE_TTL_SECONDS`
- `HFQT_SCAN_CONCURRENCY`
- `HFQT_NETWORK_INTEL_LIMIT`

### 13.3 候选新闻不够新

优先检查这四个参数：

```env
HFQT_NETWORK_INTEL_PRIMARY_AGE_HOURS=2
HFQT_NETWORK_INTEL_MAX_NEWS_AGE_HOURS=4
HFQT_NETWORK_INTEL_MAX_SOCIAL_AGE_HOURS=2
HFQT_NETWORK_INTEL_IMPORTANT_EVENT_AGE_HOURS=72
```

说明：

- 普通消息会优先取最近 `2h`，并限制在常规窗口内
- 普通新闻默认最多 `4h`
- 普通社媒默认最多 `2h`
- 只有战争、制裁、破产等重大事件，才允许在更长窗口内作为参考保留，避免两天前的一般性消息持续影响判断

### 13.4 本地模型返回了“思考过程 + JSON”

当前系统已经兼容这种输出。  
只要最终响应里能提取到合法 JSON，对分析、复核、风控链路都能继续运行。

### 13.5 为什么看板有真实新闻，但成交还是模拟的

因为当前默认 broker 是 `local_paper`。  
这代表：

- 新闻、情报、行情可以是真抓取
- 订单和盈亏仍是本地模拟

### 13.6 为什么没有下单

常见原因：

- 分析结果是 `HOLD`
- 置信度低于阈值
- review agent 否决
- risk agent 否决
- 硬规则风控拦截
- 自动交易没开
- 当前候选还没进入 `execute`

## 14. 建议的实际使用顺序

建议按这个顺序推进：

1. 先用 `local_paper + llm`
2. 跑通新闻抓取 -> 模型分析 -> 复核 -> 风控 -> 模拟成交
3. 观察几轮候选与盈亏变化
4. 再接 `futu_sim`
5. 最后再接 `uSmart`
